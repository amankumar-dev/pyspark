What is pyspark?
It is the python API for apache spark. Which allow us to write data processing applications that execute on a distributed cluster parallely. 

The Big data Problem?
Hadoop and mapreduce was invented to solve the big data storage and processing problem. 
Hadoop is a set of software libraries that are designed to fucntion over a distributed cluster of machines, Mapreduce is a data processing engine. 

The Spark Architecture?
-> Spark works in a master slave architecture. The master node is referred to as the driver and the slave nodes workers 
-> A spark session create a spark context which is an entry point to your application.
-> Spark operations(our data transformations) are executed on worker nodes.
-> Resourse on worker node and managed by a cluster manager

The resource and cluster manager?
-> The two main componets of a resource manager are the cluster manager and the worker. 
-> The cluster manager knows where the workers are located, how much memory and cpu cores each worker has.
-> When you execute a spark prog. the cluster manager determines how many work nodes will be required, how much RAM and CPY on each worker node to executed program.

The Spark Driver?
-> It is a central coordinator of spark applications. 
-> The spark driver interacts with the cluster manager to gifure out which manchines is required to run processing.
-> The spark driver requests the cluster manager to launch the spark execute to do the processing on worker nodes.
-> The spark devier creates a spark session (create a spark context) which then executes your spark configurations to allocate the required API's for your application.

The Spark Unified?
-> Apache spark provides multiple data processing components inside one single engine
-> The spark unifed stack is built on top of a strong foundation known as the spark core. 
-> The spark core provides all the necessary fuctionality to manage and run distributed applications such as scheduling and fault tollerence. 
-> The spark modules and components are built on top of spark core. 

Spark core components?
-> Spark SQL: Designed to work with databse tables and structured file formats such as csv, json, avro, orc,parquet. Can read data intor a dataframe. Can use pure SQL.
-> Spark structured streaming: Built to write applications for real time stream processing that offers high throughput and is also fault tolerant. 
-> Spark MLib: Proveds many popular machine learning algorithms out of the box.
-> Spark Graphx: Graph processing operates on data strcutres which are used to represent real life networks, such as linkedin. 

Why we use hadoop?
We use hadoop for using mapreduce which works parallely in multiple systems and for storing large data in multiple devices.

The Spark Web UI?
-> A browser dashboard that shows what your spark job is doing.
-> When we run a spark application, spark automatically starts a small web server (usually on ports 4040) and shows you: http://localhost:4040
-> It shows :-
    -> Jobs : How many jobs run / Which one succeed failed / Execution time
    -> Stages : How spark divided your job internally / DAG / Task breakdown
    -> Tasks : Duration / Failed Tasks / Shuffle read-write / CPU time
    -> Storage : Cached RDDs / Dataframes / Memory-usage
    -> Executors : Memory used / Cores used / Active Tasks
-> If wwe run multiple sparks apps , ports changes : 4040,4041,4042,...

Type of Transformations?
-> Narrow transformations (Data will not shuffle across partitions)
    Data which is in partiton complete it's operation in same partition data shouldn't show
    df.filter(df.age>50)
-> Wide transformations (Data will shuffle across partitions)
    Data moves from one partition to another and network communication happen and it's slow
    df.groupby('city').count()

What is RDD?
-> RDD stands for Resilient Distribute Dataset
    -> Resilient: Fault tolerant (agar ek node fail ho gaya, data recover ho sakta hai)
    -> Distribute: Data multiple machines / partitions me hota hai. 
    -> Dataset: Collection of Data.
-> RDD is a distributed collection of data that can be processed in parallel.
